# Chapter2

## メモ

- Rustのすべての関数は値を返す。
- 値を返せない場合はユニット型という空の値を返す。これはNullとは違う。
- Printで出力する場合はDisplayトレイトを実装している必要がある。
- Displayトレイトを実装していなくてもDebugトレイトを実装している場合は、{:?}を使用することで  
  構造体をデバッグ形式で出力することが出来る
- コマンドライン引数最初の値はプログラム自体のパスが渡される
- cargoと引数と区別したい場合は、ハイフンを二つ使用する。例：-- -n hello
-

## 依存関係について

CargoはDLしたソースをホームディレクトリの.cargoに配置します。
ビルド成果物はtarget/debug/depsに配置される。
'cargo clean'でtargetの綺麗に出来る。

## 出力内容

コマンドラインプログラムでは標準出力とエラーメッセージの2つを出力する。
深刻なエラーでない場合はそのまま処理を続行するプログラムも存在する。

## Option::unwrap()

Noneの戻り値に対してunwrapを呼び出すとパニックになる。

### ifについて

Rustのifは式である。そのためC言語など違い値を返す。
これを活用するとインラインで条件分岐して代入することが出来る。

```rust
let ending = if omit_newline {""} else {"\n"}
```

## Testについて

cargo test xxxと書くとその単語含まれるテスト関数がテストされる。

## Box

Boxを使用するとデータをスタックではなくヒープに保存することが出来る
スタックに保存されるのはヒープへポイント。
